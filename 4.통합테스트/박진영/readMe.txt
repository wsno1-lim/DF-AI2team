4. 수익률 계산 로직
 - 기간에 따라 결과가 다릅니다. 단, train 기간은 백테스팅에 포함시키면 안됩니다!
  * 전제조건 : 프로그램 실행 시 나오는 출력 참고
  * 사용자 입력값
  - 기간, 모델종류
  * 구현로직
    1. 예측-익일 주가 UP ?
     1.1. 주식 미보유 상태 ?
      -> 매수
      -> 실제 변화율 계산 : 계좌금액 UPDATE
     1.2. 주식 보유 상태 ?
      -> 유지
      -> 실제 변화율 계산 : 계좌금액 UPDATE
    2. 예측-익일 주가 DOWN ?
     2.1. 주식 미보유 상태 ?
      -> 유지
     2.2. 주식 보유 상태 ?
      -> 매도
      -> 계좌금액 => 잔고 변환
	
  * 정확도
    = 예측 정답 건수 / 시행 건수
  * 수익률
    = 현재 계좌금액 OR 잔고 / 초기자본
  * 비교
    BUY & HOLD (시작일 종가 매수, 종료 익일 종가 매도)

* 결과
  @ 정확도 : ML 약 55% , DL 약 60% 이상
    - 기간이 길 수록 일정 확률(테스트 accuracy 와 비슷)에 수렴
  @ 수익률 : 
    - 짧은 기간 대상으로는, 일반적인 변동 폭보다 큰 변동이 있을 때 해당 사건을 몇 건이나 맞추느냐에 따라 수익률이 크게 달라짐. 
      (ex : 0.5% 변동 등을 많이 맞추는 것보다 7% 변동을 한 번 맞추는게 수익률에 큰 영향을 미친다.)
    - 그러나 장기간 수행할 경우 DL이 정확도가 가장 높기 때문에 결국 수익률도 제일 높아질 것으로 예상합니다~!!
